## 任务 1：用 AI 工具搭建一个飞书机器人 Demo

请使用你最熟悉的 AI 编程工具（Claude Code / Cursor / GitHub Copilot 等），搭建一个飞书群聊机器人 demo。

### 要求

- 能接收飞书群聊消息
- 使用任意 AI 模型（Gemini / Claude / GPT 等）自动生成回复
- 代码推到你的 GitHub 仓库
- 附上 **README 文档**说明你的思路、步骤和遇到的问题

### 我们看什么

- 你用 AI 工具的**熟练程度**（不是手写代码的能力）
- GitHub 操作是否规范（commit 信息清晰、README 完整）
- 有没有**记录遇到的问题和解决过程**

> 提示：飞书开放平台提供免费的开发者账号，可以在 https://open.feishu.cn 注册。

## 任务 2：找出代码中的 Bug

本仓库中的 `buggy_bot.js` 是一段简化版的企微自动回复机器人代码。

上线后用户反馈：**「有时候机器人会重复回复同一条消息」**

### 要求

请在你的仓库中新建一个文档（如 `task2_answer.md`），提交以下内容：

1. **你找到了哪些可能导致重复回复的 Bug？**（至少找出 3 个）
2. **每个 Bug 的修复方案**（附修复后的代码片段）
3. **你的排查过程记录**：用了什么 AI 工具、问了什么问题、怎么一步步定位的

> 重点不是找到所有 Bug，而是让我们看到你**用 AI 辅助排查问题的思路和习惯**。

## 任务 3：文字问答

请在你的仓库中新建一个文档（如 `task3_answer.md`），逐题回答以下问题：

1. **AI 工具使用**：你平时用哪些 AI 工具？举一个它帮你解决实际问题的具体例子。

2. **业务理解力**：给你一个业务场景——「租户通过邮件投诉水管漏水」，请设计一个基于邮件收发的自动处理流程（从收到邮件到问题解决的完整链路，越详细越好）。

3. **Git 基础**：Git 里 rebase 和 merge 有什么区别？你一般怎么用？

4. **问题排查**：你接到一个完全不熟悉的技术需求，第一步做什么？

5. **文化认同**：你怎么理解「做了但没记录 = 没做」这句话？

6. **自驱力**：如果你 3 小时就把今天的任务做完了，接下来你会干什么？请具体描述你会做的事情，按优先顺序列出。

7. **技术与业务的关系**：假设你在工作中发现了一个很有意思的新 AI 框架，技术上看起来很强大，但公司现有的系统运行稳定、没有明显的痛点。你会怎么做？如果你的判断是「现在不引入」，你会如何处理自己对这个技术的兴趣？
